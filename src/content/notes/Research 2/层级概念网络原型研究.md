# 层级概念网络原型研究

## 研究目标

本研究的核心目标，是让模型内部的特征空间变得更加层级化、结构化，并且保持干净利落的组织形式。这里的“干净利落”指的是：特征之间的边界更清楚，概念之间的层级关系更明确，局部修改不会无序地污染其他无关区域。

通过这样的特征空间组织方式，希望进一步达到两个目标：

1. **可解释性**

   可解释性是为了让世界模型在微调过程中，能够更清楚地暴露出哪些内部概念、状态或因果因素正在影响行为结果。这样一来，奖励函数的设计就不只是依赖外部行为结果，而可以更多地参考模型内部已经形成的概念结构。

   换句话说，如果模型的特征空间是层级化和结构化的，那么我们就更容易判断：应该奖励哪一层的概念，应该惩罚哪一类状态，以及哪些特征变化只是表层现象，哪些变化才对应真正重要的能力提升。

2. **可传播干预**

   可传播干预是为了让一次有效的干预不只局限于单个样本、单个任务或单个局部行为，而是能够沿着概念网络传播到相关的上层概念、下层实例和相邻场景中。

   这种传播能力可以帮助模型举一反三：当我们在一个具体场景中完成一次微调或奖励塑形后，模型能够把这种调整迁移到结构相似的其他场景中，从而提高微调效率，减少对大量重复样本和重复标注的依赖。

因此，本研究关注的问题可以概括为：

> 如何构建一种层级概念网络，使模型的特征空间既具备可解释的结构，又支持干预效果在相关概念之间有效传播？

## 思考过程

为了让特征空间更加干净、整洁，并且具备可解释性，最先想到的方向是 **SAE（Sparse Autoencoder，稀疏自编码器）**。

SAE 的吸引力在于，它可以把模型内部复杂、混合的激活表示拆解成相对稀疏、相对独立的特征。这样一来，原本纠缠在一起的内部表示就有可能被分解成更容易解释的概念单元。因此，如果目标是提升特征空间的可解释性，SAE 是一个自然的起点。

但是，仅仅拥有一组可解释的稀疏特征还不够。本研究真正想要的是一个 **结构化的层级网络**：特征之间不只是彼此独立地存在，而是能够形成上位概念、下位概念、相邻概念和可传播关系。

最初的想法是：能不能通过设计某种 loss 函数，诱导模型在训练过程中自动形成这样的层级结构？也就是说，通过额外的正则项或结构约束，让模型的特征逐渐向层级化、网络化的方向组织。

但进一步思考后发现，单纯依靠 loss 函数诱导可能并不能真正解决问题。因为 loss 函数只是对结果施加压力，它并不一定改变特征空间本身的几何形式。如果底层特征空间仍然是普通的欧式结构，那么层级关系、概念距离和干预传播路径都可能只是被外部约束勉强拟合出来，而不是特征空间天然支持的结构。

因此，真正应该改变的不是单纯的训练目标，而是 **特征空间的形式本身**。

这个思路进一步指向了 **双曲空间（Hyperbolic Space）**。双曲空间天然适合表达层级结构：靠近中心的位置可以表示更抽象、更上位的概念，远离中心的位置可以表示更具体、更细分的实例或子概念。相比普通欧式空间，双曲空间在表达树状结构、层级网络和指数级扩展关系时更加自然。

这使得双曲空间具备了本研究想要的很多特性：

1. **层级化**

   双曲空间的几何结构天然适合承载从抽象概念到具体实例的层级关系。

2. **结构化**

   概念之间的距离、方向和位置可以被组织成更清晰的网络结构，而不只是散乱的向量分布。

3. **可传播干预**

   如果概念网络本身具有稳定的几何结构，那么对某个概念节点的干预，就有可能沿着层级边、相邻边或语义距离传播到相关概念上。

4. **更自然的可解释性**

   当特征空间的几何形式本身就对应概念层级时，解释不再只是事后分析，而可以成为特征空间组织方式的一部分。

因此，目前的思路逐渐从“用 loss 函数诱导层级结构”，转向“构建或映射到一个天然支持层级关系的特征空间”，其中双曲空间是一个非常关键的候选方向。

## 对双曲空间可实施性的疑问

但是，在进一步研究双曲空间之后，又发现一个现实问题：双曲空间虽然在表达层级结构上非常自然，但它的计算成本并不低，尤其是距离计算。

在很多双曲空间方法中，概念之间的关系需要通过双曲距离来度量。而双曲距离通常比普通欧式距离更复杂，涉及非线性函数、数值稳定性处理以及额外的几何约束。如果把这种距离计算大规模引入模型内部，尤其是在高维特征空间、海量 token 或大量概念节点上反复计算，那么工程成本和训练成本都会迅速上升。

这就带来了一个可实施性上的问题：

> 双曲空间是否只是理论上很优雅，但在真实模型微调和干预传播中成本过高？

如果双曲距离计算成为主要瓶颈，那么直接把特征空间完整地改造成双曲空间，可能并不是最现实的方案。于是需要进一步追问：

> 我们能不能绕开显式的距离计算？

更进一步说，干预传播和层级化特征空间是否真的必须依赖“距离”这个概念？

也许我们真正需要的不是完整的双曲几何，而是双曲空间所提供的某些结构性质。例如：

1. 概念之间存在上下位关系。
2. 干预可以从一个节点传播到相关节点。
3. 越抽象的概念影响范围越大，越具体的概念影响范围越局部。
4. 特征空间中存在某种可解释的组织方式，而不是无结构的向量云。

如果这些性质可以通过其他方式实现，那么距离计算可能并不是必要条件。也就是说，问题可以从：

> 如何在双曲空间中计算概念距离？

转化为：

> 如何构造一种不依赖昂贵距离计算、但仍然具备层级结构和干预传播能力的特征空间？

这个转化非常关键。它把研究重点从“使用双曲空间”推进到了“提取双曲空间中真正有用的结构思想”，然后寻找更轻量、更可实施的表达方式。

## 来自生物认知的启发

如果参考大脑中概念形成和联想扩散的方式，也许可以得到另一种启发。

大脑中的层级化特征空间，可能并不是通过显式计算每两个概念之间的几何距离来工作的。它更像是：当一个概念被激活之后，这个激活会沿着已有的神经连接向外扩散。在扩散过程中，有些相关概念会被进一步激活，有些无关或冲突的概念会被抑制。

于是整个过程不是简单的“距离越近，关系越强”，而是一个动态的激活-抑制过程：

1. 一个概念节点被初始激活。
2. 激活信号沿着关联边传播到周围节点。
3. 某些节点对当前概念形成支持，因此被进一步激活。
4. 某些节点与当前概念竞争、冲突或无关，因此被抑制。
5. 激活和抑制不断相互作用，最终形成一个稳定或半稳定的激活模式。

在这种结构中，一个概念是否“强”，不只取决于它自身的激活值，也取决于它周围网络的状态。如果它周围的相关节点大多被共同激活，那么这个概念就会变得非常强；如果它周围大多是抑制信号，或者关联节点没有被同步激活，那么这个概念的整体激活就会很弱。

这给本研究带来一个重要启示：

> 层级化特征空间和可传播干预，也许不一定需要依赖昂贵的距离计算，而可以通过概念节点之间的激活-抑制传播机制来实现。

也就是说，我们可以把“概念网络”理解为一个具有方向、强度和符号的传播系统：

1. **方向**：干预从哪个概念传播到哪个概念。
2. **强度**：传播影响有多大。
3. **符号**：传播是激活还是抑制。
4. **层级**：上位概念可以影响一组下位概念，下位概念也可以反向支持或修正上位概念。

这样一来，双曲空间提供的是一种“层级组织”的几何直觉，而大脑的激活-抑制机制则提供了一种更轻量的实现方式。我们不一定要在每次干预时计算复杂的双曲距离，而可以维护一个可解释的概念传播网络，让干预通过边权、门控和激活状态进行扩散。

因此，新的问题变成：

> 能否构建一种概念激活网络，使它既保留层级结构，又能通过激活-抑制机制实现可传播干预？

## 文献核查与思路修正

下面按照当前思考链路逐段检查：哪些已有研究已经覆盖，哪些判断需要修正，哪些地方仍然有研究潜力。

### 1. SAE 作为可解释特征空间的起点

这个判断是合理的。SAE / sparse dictionary learning 已经是当前 mechanistic interpretability 中非常核心的一条路线。

相关研究：

- [Towards Monosemanticity: Decomposing Language Models With Dictionary Learning](https://transformer-circuits.pub/2023/monosemantic-features/index.html)，Anthropic，2023。  
  这篇是 SAE 在语言模型可解释性中的关键早期工作之一，目标是把 polysemantic 的神经元表示拆成更接近单义的特征。它不是很老，而且后续发展非常快。

- [Sparse Autoencoders Find Highly Interpretable Features in Language Models](https://openreview.net/forum?id=F76bwRSLeK)，Cunningham et al.，ICLR 2024。  
  这是 SAE 路线的重要公开论文，说明 SAE 确实能在语言模型激活中找到较多可解释特征。

- [Scaling Monosemanticity: Extracting Interpretable Features from Claude 3 Sonnet](https://transformer-circuits.pub/2024/scaling-monosemanticity/index.html)，Anthropic，2024。  
  这说明 SAE / dictionary learning 已经被推到生产级模型尺度，并且能找到多语言、多模态和安全相关特征。这个方向还在发展，不是被搁置的老方向。

- [SAEBench: A Comprehensive Benchmark for Sparse Autoencoders in Language Model Interpretability](https://arxiv.org/abs/2503.09532)，Karvonen et al.，2025。  
  这篇说明领域已经进入“怎么评估 SAE 是否真的有用”的阶段。它提醒我们：重构误差、稀疏度等 proxy metric 不一定等于真实可解释性或可干预性。

需要修正的地方：

SAE 可以帮助把特征拆出来，但它本身不保证特征空间“干净、层级化、结构化”。它更像是把混合表示拆成候选概念单元。后续仍然需要解决：

1. SAE 特征是否真的单义。
2. SAE 特征之间是否存在稳定关系。
3. 对某个 SAE feature 的干预是否能可靠改变模型行为。
4. SAE feature 是否能组织成层级网络，而不是一堆扁平 feature。

因此 SAE 更适合作为“节点发现器”，不应直接等同于完整的层级概念网络。

### 2. 通过 loss 函数诱导层级结构

你对“只靠 loss 诱导可能不够”的怀疑是对的，但需要更精确：loss 诱导不是无效，而是它通常只能提供偏置，不能保证模型内部真的形成我们想要的因果结构。

相关研究：

- [Matryoshka Representation Learning](https://arxiv.org/abs/2205.13147)，Kusupati et al.，2022。  
  这篇提出让 embedding 的前若干维表达粗粒度信息，更多维度表达细粒度信息。它不是双曲空间路线，但很接近“粗到细、层级化表示”的目标，而且工程上比较轻量。后续仍在发展。

- [MIPIC: Matryoshka Representation Learning via Self-Distilled Intra-Relational and Progressive Information Chaining](https://arxiv.org/abs/2604.24374)，2026。  
  这是较新的 Matryoshka 表示学习扩展，说明“嵌套式、粗到细表示”仍然是活跃方向。

- [MatryoshkaLoRA: Learning Accurate Hierarchical Low-Rank Representations for LLM Fine-Tuning](https://arxiv.org/abs/2605.07850)，2026。  
  这把 Matryoshka 思路用于 LoRA 微调，和“提高微调效率”这条目标很相关。

可以补充进当前方案的想法：

loss 函数不应该作为唯一机制，但可以作为辅助约束。例如：

1. 用 SAE 找出候选概念节点。
2. 用层级/嵌套式 loss 让表示具备粗到细结构。
3. 用干预实验检验这些结构是否真的有因果作用。

换句话说，不是“loss vs 特征空间形式”二选一，而是：特征空间结构是主体，loss 是塑形工具，干预验证是验收标准。

### 3. 双曲空间作为层级结构的几何候选

这个判断也有扎实文献基础。双曲空间确实天然适合表达树状结构、taxonomy 和层级关系。

相关研究：

- [Poincare Embeddings for Learning Hierarchical Representations](https://arxiv.org/abs/1705.08039)，Nickel & Kiela，2017。  
  这是双曲嵌入的经典论文。它已经比较早，但不是“死方向”，因为后续大量工作继续发展了这条线。

- [Hyperbolic Neural Networks](https://arxiv.org/abs/1805.09112)，Ganea et al.，2018。  
  这篇把双曲几何扩展到神经网络层，属于双曲深度学习的基础工作。

- [Hyperbolic Deep Learning in Computer Vision: A Survey](https://arxiv.org/abs/2305.06611)，Mettes et al.，2023。  
  说明双曲学习已经扩展到视觉、鲁棒性、少样本和不确定性等方向。

- [Hyperbolic Deep Learning for Foundation Models: A Survey](https://arxiv.org/abs/2507.17787)，He et al.，2025。  
  这是较新的综述，说明双曲空间正在和 LLM、VLM、多模态 foundation model 结合，仍然有潜力。

需要修正的地方：

双曲空间不只是“层级化”，它同时编码了两类信息：

1. 半径方向更适合表达抽象程度、层级深度、popularity。
2. 角度方向更适合表达同层语义相似性。

所以如果未来借鉴双曲空间，不一定要完整搬运距离公式，而可以借鉴“径向层级 + 角向语义分区”的分解思想。

### 4. 双曲距离计算成本和可实施性

你的担忧是合理的。双曲空间的指数、反双曲函数、数值稳定性、流形约束、切空间映射，都会带来额外工程成本。尤其如果做全量 pairwise distance，成本会非常高。

但需要补充一个边界：双曲方法并不一定必须做全量距离计算，也不一定只能用最原始的 Poincare ball 公式。

相关研究：

- [Towards Scalable Hyperbolic Neural Networks using Taylor Series Approximations](https://arxiv.org/abs/2206.03610)，Choudhary & Reddy，2022。  
  这篇直接针对双曲网络的可扩展性问题，用 Taylor 近似减少昂贵的双曲函数计算。

- [Fully Hyperbolic Neural Networks](https://arxiv.org/abs/2105.14686)，Chen et al.，2021。  
  这篇尝试避免频繁在欧式切空间和双曲空间之间来回映射，说明“混合空间计算”本身就是一个痛点。

- [Lorentzian Residual Neural Networks](https://arxiv.org/abs/2412.14695)，He et al.，2024。  
  这说明 Lorentz 模型在稳定性、残差连接和深层网络上仍有改进空间。

- [Fast and Geometrically Grounded Lorentz Neural Networks](https://arxiv.org/abs/2601.21529)，van der Klis et al.，2026。  
  这是很新的工作，目标就是缩小双曲网络和欧式网络之间的计算差距。说明“降低双曲计算成本”仍是活跃问题，而不是已经解决或被放弃。

因此更准确的判断是：

> 双曲空间有理论吸引力，但完整双曲化可能成本过高；更现实的路线是只吸收它的层级归纳偏置，或者使用近似、Lorentz 模型、局部邻域传播、缓存和稀疏图来降低计算成本。

### 5. 干预传播是否必须依赖距离

这个问题非常关键，而且你的直觉是对的：不一定需要。

“距离”只是表达相关性的一种方式。干预传播也可以基于显式图结构、边权、门控、注意力、因果边或 learned routing 来完成。也就是说，传播需要的是“可路由的关系”，而不是一定要有“几何距离”。

相关研究：

- [Steering Language Models With Activation Engineering](https://arxiv.org/abs/2308.10248)，Turner et al.，2023。  
  这篇用 activation addition 直接在模型内部添加 steering vector，不依赖双曲距离，说明可干预性可以通过方向和激活修改实现。

- [Towards Best Practices of Activation Patching in Language Models](https://arxiv.org/abs/2309.16042)，Zhang & Nanda，2023 / ICLR 2024。  
  这篇系统讨论 activation patching，说明内部激活干预已经是 mechanistic interpretability 的标准工具之一。

- [Circuit Tracing / On the Biology of a Large Language Model](https://transformer-circuits.pub/2025/attribution-graphs/biology.html)，Anthropic，2025。  
  这非常接近你的想法：用 attribution graph 把特征当作节点，把因果影响当作边，然后通过抑制或激活特征组观察传播效果。这个方向非常新，而且明显还在发展。

可以补充进当前方案的想法：

与其问“两个概念距离多远”，不如问：

1. A 概念被激活后，会直接推动哪些概念？
2. 会抑制哪些概念？
3. 这些影响是一跳有效，还是多跳有效？
4. 干预传播后，模型输出是否按预期变化？
5. 传播路径是否稳定跨样本、跨任务复现？

这比单纯距离更接近“可传播干预”的核心。

### 6. 大脑中的激活-抑制网络启发

这个类比有依据，但需要避免过度生物化。大脑确实有 spreading activation、竞争、抑制、吸引子动力学等机制；不过我们不能直接断言大模型内部就是这样运作。

相关研究：

- Collins & Loftus 的 spreading activation 理论，1975。  
  这是语义网络中“一个概念激活后向相关概念扩散”的经典认知科学模型。它很老，但作为思想源头仍然重要。

- ACT-R / spreading activation memory models，Anderson 等，1990s-2000s。  
  这些认知架构中，记忆检索和概念联想都依赖激活强度、连接强度和衰减机制。方向比较老，但没有直接成为现代 LLM 的主流工程路线。

- Hopfield networks / modern Hopfield networks。  
  吸引子网络提供了“多个节点互相支持，最后收敛到稳定模式”的数学框架。传统 Hopfield 网络很老，但 modern Hopfield networks 和 attention 的联系让它重新有了现代意义。

- Graph Neural Networks / Message Passing Neural Networks。  
  GNN 的核心就是节点之间按边传播消息。虽然它不是专门为可解释性设计的，但非常适合作为“概念节点之间传播激活/抑制”的工程模板。

需要修正的地方：

“周围都是激活，所以这个神经元特别强；周围都是抑制，所以它很弱”这个直觉是合理的，但需要形式化。更准确地说，一个概念节点的激活可以写成：

```text
new_activation = self_state + excitatory_messages - inhibitory_messages + external_intervention
```

然后再通过门控、归一化、阈值或非线性函数决定是否继续传播。

这给当前研究带来一个很好的工程方向：构建一个 signed concept graph，其中边可以是正的激活边，也可以是负的抑制边。这样就能绕开双曲距离，直接实现可解释的传播机制。

## 当前思路的主要风险

1. **可解释性不等于可控性**

   一个 feature 能被解释，不代表它一定是好的控制手柄。2025 年之后的 SAE 评估研究反复提醒：interpretability metric 和 steering utility 之间可能不完全一致。

2. **传播不等于泛化**

   干预能传播到邻居节点，不代表它一定能举一反三。需要验证传播后的行为是否在新任务、新语境中仍然正确。

3. **层级结构可能是人为投影**

   如果层级网络是后验构造出来的，而不是模型真实使用的结构，那么它可能看起来漂亮，但干预时不稳定。

4. **脑启发不能直接当作证据**

   生物认知可以提供结构灵感，但最终仍然要靠模型内部干预实验验证。

## 可以补充的原型路线

目前最有潜力的路线不是“纯 SAE”或“纯双曲空间”，而是一个混合原型：

1. **SAE / transcoder 提取候选概念节点**

   用 SAE 找 feature；如果目标是传播和因果图，可以进一步关注 Anthropic 2025 年 circuit tracing 中的 transcoder / cross-layer transcoder 思路。

2. **构建 signed concept graph**

   节点是可解释 feature，边表示激活或抑制关系。边可以来自：

   - activation patching；
   - attribution graph；
   - feature co-activation / anti-correlation；
   - 下游输出变化；
   - 人工先验 taxonomy。

3. **引入轻量层级坐标**

   不一定完整使用双曲距离，但可以保留类似双曲空间的结构：

   - `level` 表示抽象层级；
   - `cluster` 表示语义领域；
   - `scope` 表示干预影响范围；
   - `polarity` 表示激活或抑制。

4. **传播规则替代距离计算**

   干预时不计算全局距离，而是在图上做有限步传播：

```text
activation[t + 1] = gate(edge) * weight(edge) * activation[t]
```

   只传播 1-3 跳，并通过阈值截断弱信号，避免成本失控。

5. **用行为实验闭环验证**

   每次干预后都检查：

   - 目标行为是否改变；
   - 非目标行为是否保持；
   - 相关任务是否迁移；
   - 无关任务是否不受污染；
   - 多跳传播是否真的存在。

## 阶段性判断

你的思考主线总体是成立的，而且和 2023-2026 年 interpretability、SAE、activation steering、circuit tracing、hyperbolic foundation model 这些活跃方向有明显交集。

最需要修正的是：不要把“层级结构”完全押在双曲空间上，也不要把“可解释 feature”直接等同于“可传播干预节点”。更稳妥的研究问题应该改写为：

> 如何从模型内部提取可解释概念节点，并构建一个带有层级、方向、强度和正负符号的概念传播图，使干预能够在图上有限步传播，并通过行为实验验证其泛化效果？

这个版本比“构建双曲特征空间”更工程可行，也更接近当前最新的 mechanistic interpretability 发展方向。

## GNN 与 Transformer 的结合方式

在进一步思考“概念网络如何实现可传播干预”时，一个关键问题是：GNN 和传统 Transformer 的向量相似度机制之间是什么关系？它们是否可以结合？

这里可以先把二者的定位区分清楚：

> Transformer 更擅长在欧式向量空间中进行全局、动态、隐式的关系建模；GNN 更擅长在显式图结构上进行局部、可追踪、结构化的消息传播。

因此，GNN 不一定要替代 Transformer。更有潜力的方向，是让 Transformer 负责强表达和概念提取，让 GNN 负责结构组织和干预传播。

### 1. Transformer / 欧式向量相似度路线

传统 Transformer 通常把 token、patch 或概念表示成高维向量，然后通过点积、余弦相似度或 attention score 来判断不同位置之间的相关性。

典型形式是：

```text
attention(Q, K, V) = softmax(QK^T / sqrt(d)) V
```

也就是说，每个位置拿自己的 query 去和其他位置的 key 计算相似度，再按照相似度加权聚合 value。

优点：

1. **全局建模能力强**

   每个 token 理论上可以直接关注所有其他 token，不需要预先定义图结构。

2. **表达能力强**

   关系可以随上下文动态变化，不固定在某一张显式图上。

3. **适合大规模预训练**

   当前 LLM 的成功主要来自这种可扩展的 Transformer 架构。

4. **端到端训练简单**

   不需要提前定义节点、边、层级关系或传播规则。

缺点：

1. **关系不够显式**

   attention 高不一定代表真正的因果关系，也不一定代表可解释的概念关系。

2. **可解释性较弱**

   高维向量空间中的相似度，很难直接解释成“上位概念”“下位概念”“激活边”或“抑制边”。

3. **干预传播不好控制**

   如果想让一个概念激活后只影响某些相关概念、抑制某些冲突概念，纯 Transformer 空间很难提供清晰的传播路径。

4. **计算成本高**

   标准 attention 对序列长度是二次复杂度，即 `O(n^2)`。当节点或 token 数量很大时成本明显上升。

潜力判断：

Transformer 仍然是底层表达能力最强的主干结构，短期内不适合被 GNN 完全替代。但如果目标是可解释性和可控干预，单独依赖 Transformer 的隐式相似度是不够的。

### 2. GNN 路线

GNN 的核心机制是 message passing，即消息传递。图中的每个节点从邻居节点接收信息，再更新自己的状态。

基本形式可以写成：

```text
node_new = update(node_old, aggregate(messages from neighbors))
```

如果放到概念网络中，可以理解为：

```text
概念新激活值 = 自身激活 + 邻居激活输入 - 邻居抑制输入
```

优点：

1. **结构显式**

   节点可以对应概念或 SAE feature，边可以对应激活、抑制、层级、相邻、因果或共现关系。

2. **传播路径可追踪**

   干预从哪个节点出发，经过哪些边，影响哪些节点，都可以被记录和解释。

3. **适合层级网络**

   树、DAG、知识图谱、概念图、因果图都可以自然表示成图结构。

4. **稀疏计算更便宜**

   如果图是稀疏的，每个节点只和少数邻居交互，不需要像 attention 那样做全局两两相似度计算。

缺点：

1. **依赖图质量**

   如果节点或边构造错了，传播机制会放大错误结构。

2. **长距离传播困难**

   普通 GNN 需要堆叠多层才能传播到远处节点，容易出现 oversmoothing 和 oversquashing。

3. **上下文灵活性较弱**

   如果图结构过于固定，就不如 Transformer 那样能根据上下文动态调整关系。

4. **单独作为大模型主干不够现实**

   对开放语言建模而言，纯 GNN 很难替代 Transformer 的大规模序列建模能力。

潜力判断：

GNN 更适合作为概念网络、知识结构、可解释传播层，而不是直接替代 LLM 主干。对于本研究，它的最大价值是提供“可传播干预”的工程模板。

### 3. 结合方式一：Graph Transformer

Graph Transformer 是把图结构注入 Transformer。它仍然使用 attention，但 attention 会受到图邻接关系、边类型、图距离或结构编码的影响。

可以理解为：

```text
Transformer 的全局注意力 + 图结构偏置
```

优点：

1. 保留 Transformer 的全局表达能力。
2. 引入图结构后，模型更容易理解节点关系。
3. 适合知识图谱、分子图、程序图、结构化推理等任务。
4. 可以把边类型、层级深度、最短路径等结构信息加入 attention bias。

缺点：

1. 计算成本仍然可能接近 Transformer。
2. 图结构只是作为 bias 注入，不一定形成真正可解释的传播路径。
3. 如果 attention 是全连接的，图结构约束可能被模型绕开。
4. 对本研究而言，它更像是“结构增强 Transformer”，不是最直接的干预传播机制。

潜力判断：

Graph Transformer 是很有发展潜力的通用方向，尤其适合图基础模型和结构化推理。但如果目标是“干预传播路径可解释”，还需要额外的因果验证或显式传播模块。

### 4. 结合方式二：GNN + Transformer 双模块架构

这种方式把 GNN 和 Transformer 分成两个模块：

```text
GNN 负责局部结构传播
Transformer 负责全局整合
```

或者反过来：

```text
Transformer 负责提取上下文表示
GNN 负责在结构图上组织和传播
```

优点：

1. 两者分工清楚。
2. GNN 可以处理显式结构，Transformer 可以处理上下文和全局依赖。
3. 比纯 Graph Transformer 更容易控制传播路径。
4. 可以根据任务决定先 Transformer 后 GNN，还是先 GNN 后 Transformer。

缺点：

1. 架构复杂度更高。
2. 两个模块之间如何对齐表示是难点。
3. 如果联合训练，调参和稳定性会更复杂。
4. 如果图结构动态变化，系统实现成本会升高。

潜力判断：

这是一条工程上可行、研究上也有空间的路线。对于本研究，更适合采用“Transformer 提取特征，GNN 组织传播”的方向，而不是让 GNN 先处理原始 token。

### 5. 结合方式三：Concept Graph over Transformer Features

这是最贴近本研究目标的路线。

基本思路是：

```text
LLM / Transformer hidden states
        ↓
SAE / transcoder 提取可解释 feature
        ↓
把 feature 当作概念节点
        ↓
构建 signed hierarchical concept graph
        ↓
用 GNN / message passing 做激活-抑制传播
        ↓
把传播结果用于 steering、reward design 或 fine-tuning
```

优点：

1. **不需要重训整个 Transformer**

   可以在已有模型之上构建解释层和干预层。

2. **兼容 SAE / transcoder**

   SAE 负责发现候选概念节点，GNN 负责组织这些节点之间的传播关系。

3. **最适合可解释干预**

   每个节点、每条边、每次传播都可以被记录和检查。

4. **可以绕开昂贵距离计算**

   干预时不需要计算全局双曲距离，而是在图上做有限步传播。

5. **可以同时支持激活和抑制**

   通过 signed edge 表示正向激活和负向抑制。

缺点：

1. 概念节点质量依赖 SAE / transcoder 的质量。
2. 边的构造是关键难点，需要 activation patching、attribution graph、共激活统计或人工先验辅助。
3. 图可能随模型层、上下文、任务而变化，不一定有一个全局固定图。
4. 传播是否真的提高泛化，需要严格行为实验验证。

潜力判断：

这是当前最适合本研究的路线。它把 Transformer 的强表达能力、SAE 的可解释 feature、GNN 的传播机制结合起来，同时避免把整个模型改造成双曲空间或纯图模型。

### 6. 结合方式四：Graph-Augmented RAG / Knowledge Graph + LLM

这种方式不一定直接改模型内部，而是在外部引入知识图谱或概念图，帮助 LLM 检索、推理和生成。

基本形式是：

```text
用户问题
  ↓
知识图谱 / 概念图检索相关节点和路径
  ↓
把结构化上下文交给 LLM
  ↓
LLM 生成回答或决策
```

优点：

1. 工程上更容易落地。
2. 不需要访问模型内部激活。
3. 图谱路径可以提供一定可解释性。
4. 适合知识密集型问答、医学、法律、企业知识库等场景。

缺点：

1. 它主要增强外部知识，不一定改变模型内部特征空间。
2. 可传播干预更多发生在外部图谱，不一定能改变模型内部概念表示。
3. LLM 是否真正遵循图谱路径仍然需要验证。
4. 对本研究的“内部特征空间层级化”目标帮助有限。

潜力判断：

这条路线应用潜力很强，但更偏产品和知识系统。如果研究目标是模型内部可解释性和微调效率，它只能作为辅助，不是核心方案。

### 7. 结合方式五：Graph Foundation Model + LLM

Graph Foundation Model 是较新的方向，目标是像 LLM 一样预训练一个可迁移的通用图模型。它可以和 LLM 结合，用于处理知识图谱、分子图、社交图、推荐图、程序图等结构化数据。

相关新进展包括：

- [Graph Foundation Models: A Comprehensive Survey](https://arxiv.org/abs/2505.15116)，2025。
- [Billion-Scale Graph Foundation Models](https://arxiv.org/abs/2602.04768)，2026。
- [Toward a universal foundation model for graph-structured data](https://arxiv.org/abs/2604.06391)，2026。

优点：

1. 有机会形成通用的图结构表示能力。
2. 可以和 LLM 互补：LLM 处理语言，GFM 处理图结构。
3. 对跨图迁移、少样本图任务、结构化推理有潜力。
4. 说明 GNN / 图模型方向仍在发展，并没有被 Transformer 完全取代。

缺点：

1. 目前还没有像 LLM 那样成熟的统一范式。
2. 不同图之间的节点、边、特征空间差异很大，迁移难度高。
3. 训练和评估标准还在形成中。
4. 对本研究而言，可能过重，不适合作为第一版原型。

潜力判断：

这是长期方向，值得关注。但当前更适合作为背景趋势，而不是马上采用的技术核心。

### 8. 对本研究的阶段性判断

对于“层级概念网络原型研究”，最现实的结合方式不是让 GNN 替代 Transformer，也不是直接训练一个庞大的 Graph Foundation Model，而是采用：

> Transformer / LLM 作为底层表达系统，SAE / transcoder 作为概念提取工具，GNN / message passing 作为概念传播与干预层。

也就是：

```text
Transformer 负责表达
SAE 负责拆解
Graph 负责组织
GNN 负责传播
Intervention 负责验证
```

这种路线的优势是：

1. 不需要从零训练大模型。
2. 不需要完整双曲空间距离计算。
3. 可以保留 Transformer 的强表达能力。
4. 可以引入显式层级结构和正负传播边。
5. 每次干预都有路径可追踪，适合可解释性研究。

最关键的风险是：

1. SAE feature 不一定是真正稳定的概念节点。
2. 图边不一定代表真实因果关系。
3. 传播后得到的变化不一定能泛化。
4. 过度结构化可能损害模型原本灵活的上下文能力。

因此，第一版原型应该保持轻量：

1. 先选少量可解释 feature 作为节点。
2. 用 activation patching 或 attribution graph 建少量高置信边。
3. 区分激活边和抑制边。
4. 只做 1-3 跳有限步传播。
5. 用行为变化和 off-target 副作用验证干预是否有效。

这样可以在不大规模改造模型的情况下，验证“层级概念网络 + 可传播干预”这条路线是否真的成立。
