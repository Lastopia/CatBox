## Latent Space
type: 关键词

他和以下几种概念相关。
- State space 真实环境状态空间。是一种理想的概念。
- Observation space 观测空间，有时候也就叫做pixel space。

latent space 被定义为被训练成的可模拟世界演化的空间。

latent state 通常是 feature vector 的一种，但 feature vector 不一定是 latent state，一个特征变量需要联系模型的上下文来确定他学的这个特征变量是不是就是潜空间变量。

最初被 PlaNet 选择。理由如下：
-  比 observation space 更加低维
-  比 state space 更加可实现（有限的信息，比如只有视觉模块）
-  比 model-free 更加样本高效
-  比 symbolic space 更加容易端到端学习


## POMDP
type: 关键词

**Partially observable Markov decision process**
部分可观测马尔科夫决策过程
- 普通markov假设智能体的每一步都能看到完整状态，比如位置，速度，目标位置等。但是目前只能处理图像，所以不是环境真实状态。所以问题变成了如下：
	- 当前真实状态由上一个真实状态和上一个动作决定
	- 当前图像由当前真实状态生成
	- 当前奖励也由当前真实状态决定
	- 智能体的动作只能依赖过去和现在看到的图像、过去执行过的动作


## World Models

type: 论文
date: 2018
authors: David Ha et al.
link: https://arxiv.org/abs/1803.10122

### 关键点
- 无监督快速训练来学习环境的压缩时空表征
- 人类对世界的认知只是一种简化的概念关系模型
- 模型目的是根据当前感官数据预测未来，并根据需要采取行动
- RL存在信用分配问题，即奖惩机制难以归因，网络规模小
- 采用成本较小的虚拟环境代替现实环境进行训练（比如赛车游戏）
- 驱动模型的不再是更小的 loss， 而是预测的未来的期望值
- 计算期望值需要设计奖励函数来诱导AI的行为，就像给狗的骨头一样
- 尝试设计还原人类好奇心与内在动机的奖励函数
- 人在睡觉的时大脑会重放近期记忆，一定程度后该记忆会逐渐脱离海马体独立存在

## KV Cache
type: 关键词

LLM (Transformer架构) 一次生成过程包含`Prefill`& `Decode` 两个阶段。
1. Prefill 是将prompt输入，产生一次性Q矩阵，然后计算增量KV Cache， 注意，成为KV Cache后就会被冻结，然后Q用完后直接抛弃。
2. Decode是一个一个token生成答案的过程，和Prefill一样，只不过是一行一行的Q，一行一行的KV Cache 增量。Q也是不保留的。

> 和位置编码的外推性的关系？

KV Cache 是决定 LLM 上下文长度的工程限制，决定算不算的动。
位置编码外推性是 LLM 建模能力限制，决定算的准不准。

## VAE
type: 关键词

Variational Autoencoder，变分自编码器，为了`生成`。类似 SAE， 先 Encoder 将输入编码成`潜空间概率分布`，根据分布采样 z，再用 z 重建。过程有损。

> 和SAE的区别

SAE 稀疏自编码器是为了`解释`, 直接编码成 z， 再根据 z 重建，只不过会用`正则化`让 z `稀疏`

## Image Generators are Generalist Vision Learners

type: 论文
date: 2026
authors: Valentin Gabeur, Shangbang Long, Songyou Peng, Paul Voigtlaender, Shuyang Sun, Yanan Bao, Karen Truong, Zhicheng Wang, Wenlei Zhou, Jonathan T. Barron, Kyle Genova, Nithish Kannen, Sherry Ben, Yandong Li, Mandy Guo, Suhas Yogin, Yiming Gu, Huizhong Chen, Oliver Wang, Saining Xie, Howard Zhou, Kaiming He, Thomas Funkhouser, Jean-Baptiste Alayrac, Radu Soricut
link: https://arxiv.org/abs/2604.20329

### 核心问题

这篇论文想回答一个很直接但很重要的问题：图像生成模型是不是只会“画图”，还是它在大规模生成式预训练中已经学到了足够强的视觉理解表征？

作者的判断是：图像生成训练在视觉领域里可能扮演了类似 LLM 生成式预训练在语言领域里的角色。也就是说，模型为了生成高质量、语义可控、几何合理的图像，必须内化大量关于物体、空间、语义、尺度和关系的知识。这些知识不一定直接以分类器或分割器的形式暴露出来，但可以通过轻量 instruction-tuning 被“解锁”为通用视觉任务能力。

### 方法

作者基于 Nano Banana Pro 做轻量 instruction-tuning，得到一个通用视觉模型 Vision Banana。关键设计不是给每个任务加专门 head，也不是为深度、分割、法线估计分别设计架构，而是把视觉理解任务的输出统一参数化为 RGB 图像。

这意味着：

- 语义分割可以输出一张彩色 mask 图，不同类别对应不同颜色；
- 实例分割可以让同一类别的不同实例用不同颜色表示；
- 深度估计可以通过可逆的 false-color 映射，把真实 metric depth 编码成 RGB，再从生成结果反解回深度；
- 表面法线天然可以映射到 RGB 通道，因此也能被看成图像生成任务。

这个设计的重点在于：感知任务被重新写成“生成一张符合约束的图像”。模型只需要学会按照指令生成可解码的视觉输出，底层表征仍然来自原本的图像生成预训练。

### 实验结果

Vision Banana 在多个 2D 和 3D 视觉理解任务上达到或接近 SOTA：

- 在 Cityscapes 语义分割上，Vision Banana 的 mIoU 为 0.699，高于 SAM 3 的 0.652。
- 在 RefCOCOg referring segmentation 上，Vision Banana 的 cIoU 为 0.738，略高于 SAM 3 Agent 的 0.734。
- 在 ReasonSeg 上，Vision Banana 的 gIoU 为 0.793，高于 SAM 3 Agent 的 0.770。
- 在 metric depth estimation 上，作者报告 Vision Banana 在多个公开数据集的平均表现超过 Depth Anything 3、UniK3D、MoGe-2 等专门模型，并且训练和推理都不依赖相机内参。
- 在 surface normal estimation 上，Vision Banana 在室内数据集平均指标上优于或接近 Lotus-2 等专门方法。
- 在图像生成和图像编辑能力上，Vision Banana 基本保留了 Nano Banana Pro 的能力，没有明显灾难性遗忘。

这些结果支持作者的核心论断：生成式视觉预训练本身已经学到了通用视觉理解能力，instruction-tuning 更像是把这些能力对齐到可评测、可调用的输出格式。

### 关键贡献

第一，论文把“图像生成模型具有视觉理解能力”从定性观察推进到了一组可量化 benchmark。之前很多工作发现生成模型可以 zero-shot 画出类似深度图、分割图的结果，但难以严格解码和评测；这篇通过 RGB 输出协议把结果变成可计算指标。

第二，它提出了一个很强的统一接口观点：文本生成可以统一语言任务，图像生成也可能统一视觉任务。只要输出能被编码成 RGB，很多视觉理解任务都可以转化为生成任务。

第三，它削弱了“视觉理解必须依赖专门判别式架构”的假设。分割、深度、法线估计过去往往需要专门 head、专门 loss、专门数据；这里作者展示了一个生成模型通过轻量对齐也可以做到很强。

### 与当前研究思路的关系

这篇对“层级概念网络 / 可解释特征空间”的启发很大。它说明生成模型内部可能已经形成了跨任务的视觉世界模型，只是这些表征并不会自动以我们想要的结构暴露出来。instruction-tuning 的作用类似于给内部表征增加一个可读接口。

可以把它和当前思路对应起来：

- 生成式预训练提供强大的底层表征；
- instruction-tuning 提供任务接口和输出协议；
- 如果进一步加入 SAE / transcoder / concept graph，就可能把这些隐含视觉概念拆出来，组织成可解释、可传播、可干预的概念网络；
- RGB 输出协议本身可以看成一种“外部可解码表征”，但它还不是内部概念结构，需要进一步做机制解释或特征提取。

因此，这篇论文更像是在证明：大生成模型内部确实有值得挖的通用视觉表征。下一步问题不是“它有没有理解能力”，而是“这些能力在内部如何组织，以及能不能被可控地解释和干预”。

### 局限

第一，Vision Banana 依赖强大的闭源基础生成模型 Nano Banana Pro，因此很难判断哪些能力来自预训练数据规模、哪些来自架构、哪些来自 instruction-tuning。

第二，论文证明的是轻量对齐后可以在 benchmark 上表现很好，但不等于已经解释了模型内部机制。它没有直接回答内部表征是否层级化、是否可组合、是否存在稳定概念节点。

第三，把所有输出都转成 RGB 很优雅，但也带来解码协议问题。深度估计需要设计可逆颜色映射，实例分割需要颜色聚类，这些工程选择会影响评测稳定性。

第四，作者也承认生成模型推理成本显著高于轻量专门模型，这会限制实际部署。

### 精读判断

这篇值得精读。它和当前“可解释概念网络”的关系不是方法上的直接替代，而是提供了一个重要前提：生成式视觉模型可能已经拥有跨任务、跨语义、跨几何的内部表征。真正值得追的是如何把这些表征抽取为稳定概念节点，并进一步建立层级关系、传播关系和干预路径。

可以重点读三块：

1. Method 里 RGB 输出协议如何把不同视觉任务统一成生成任务。
2. 2D / 3D benchmark 里哪些任务真正支持“通用视觉学习者”的结论。
3. Discussion 里关于 image generation as universal interface 的论证，因为它和“生成作为统一接口”这个大方向高度相关。
