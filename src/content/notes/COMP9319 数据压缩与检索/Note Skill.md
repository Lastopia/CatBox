# File Shape

- Frontmatter 有以下属性：
	- `date`
	- `description`
	- `coreIdeas`
- 正文用概念块组织，而不是按 slide 页码组织。
- 标题层级优先使用：
	- `#`：大主题 / Part。
	- `##`：核心概念块。
	- `**标题**`：局部小主题或强调。
	- `-`：具体知识点。

# Writing Style

- 以中文解释为主，关键英文术语保留原词。
- 关键词第一次出现时，使用中文 + 英文，例如 命题逻辑(propositional logic)。
- 后续再次出现时，可以直接用中文或反引号英文，例如 `Bayesian network`。
- 语气保持“自己讲给自己听”的复习风格：直观、短句、能解释为什么重要。
- 每个 bullet 尽量只承载一个知识点。
- 只使用一种 bullet 符号：统一用 `-`，不要混用 `*`、`+` 或其他 bullet。
- 正文里的 bullet 层级用真实 tab 缩进；子 bullet 写成 `	-`，不要用两个空格写成 `  -`。
- 保留少量必要公式，例如 $P(A \mid B)$；公式后面要直接解释含义，不要写“直觉是”这类提示词。
- 如果一个 bullet 只有一个公式或一个短符号说明，优先合成一行，不要拆成子 bullet。
	- 推荐：`- 一个事件和它的 negation 互补：$P(\alpha)+P(\neg\alpha)=1$。`
	- 避免：
		- `- 一个事件和它的 negation 互补：`
		- `  - $P(\alpha)+P(\neg\alpha)=1$`
- 解释后面直接接公式时，用中文冒号 `：` 引出公式；不要用逗号把公式硬接在句尾。
- 用引用块 `>` 写核心 takeaway、关键对比、考试复习时最该记住的一句话。
- 如果课件原句特别关键，可以保留英文原句，并补一句中文解释。
- 不写过度教材化的定义，不写冗长背景故事。

# Content Rules

- 先抓主线，再补细节：
	1. 这节课要解决什么问题。
	2. 每个概念为什么被引入。
	3. 这个概念和前后内容是什么关系。
- 对每个重要概念至少写清：
	- `是什么`
	- `为什么需要`
	- `如何使用 / 后面怎么用到`
- 课件里的列表要压缩，但不能漏掉：
	- 分类维度
	- 核心批评
	- 主要模型
	- 关键人物
	- 关键算法
	- 重要公式或符号
- 例子只保留能帮助理解抽象概念的部分。
- 例子变量可以列出，但不要复述整段故事。
- 复杂公式如果需要例子说明，优先自己构造极简场景和小数字，不照搬课件里的复杂表格。
- 例子服务于理解公式：数字要少、计算要短、变量名要贴近日常语境。
- 小数统一写前导 0，例如 `0.3`、`0.05`，不要写 `.3`、`.05`。
- 无视 `[[ ]]` 包裹的图片或 Obsidian 链接内容，除非用户明确要求处理图片。

# Preferred Patterns

- 对比类内容使用并列结构：
	- `Logic vs Probability`
	- `Directed / Undirected`
	- `Static / Dynamic`
	- `Probabilistic / Decisional`
- 历史或动机类内容写成因果链：
	- 逻辑推理的限制 -> degree of belief -> probability calculus -> PGM。
- 形式化内容先给直观解释，再给符号：
	- 先说 “world 是一个完整状态”。
	- 再写 $w \models \alpha$ 表示 sentence 在 world 中为真。
- 与后续课程相关的桥梁要写出来：
	- `sentence = event`
	- `world = sample point`
	- `models = satisfying worlds`

# Concision Standard

- 删除空泛句，例如“这个非常重要”，除非后面说明重要在哪里。
- 删除机械翻译，保留复习时真正需要的逻辑。
- 避免为了公式单独开一层列表；只有多个并列公式、多个条件或多个 case 时才使用子 bullet。
- 简短解释、术语和公式能放在一行时就放在一行，避免把一个知识点误拆成两种层级。
- 一节 lecture 的最终笔记应当完整但轻量；读者应能在几分钟内扫完主线。
