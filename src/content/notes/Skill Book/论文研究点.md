# 适用目标

这个 skill 用来把一篇或一组论文整理成“论文研究点”笔记。目标不是写论文摘要，也不是做文献综述模板，而是把论文放进自己的研究问题里，提炼出：

- 这篇论文解决了什么问题
- 它为什么对当前研究路线有价值
- 它引入了哪些关键概念
- 它的方法结构是什么
- 它有什么局限
- 它能把当前研究问题推进到哪里

写作时优先服务自己的研究判断。不要把笔记写成论文复述。

# 文件组织

一个研究笔记本目录下至少包含两类 Markdown：

```text
研究笔记1/
  世界模型总览.md
  研究计划总览.md
  _链接卡片.md
  附件/
    Pasted image ...
```

普通笔记负责正文，例如 `世界模型总览.md`、`研究计划总览.md`。

`_链接卡片.md` 负责维护所有可点击的论文卡片和关键词卡片。

图片统一放在同级目录下的 `附件/` 文件夹，并在 Markdown 中使用：

```md
![[Pasted image 20260603083343.png]]
```

# 普通笔记 frontmatter

普通笔记只保留必要属性：

```md
---
date: 2026-06-02
description: 世界模型的研究总览
coreIdeas:
  - 这里写最重要的研究判断
---
```

不要写 `title`。页面标题直接使用文件名。

不要写 `notebook`。笔记所属笔记本直接由文件夹目录决定。

# 正文组织方式

正文用一级标题组织研究路线或研究问题。

推荐结构：

```md
# 论文精读路线

1. 最早的概念来自 [World Models](https://arxiv.org/abs/1803.10122)
2. 后续路线来自 [Learning Latent Dynamics for Planning from Pixels](https://arxiv.org/abs/1811.04551)

# 当前研究问题

这里写这组论文对当前问题的意义。
```

如果是围绕一个想法展开，不必强行按论文逐篇排序，可以按问题推进：

```md
# 对世界模型的理解

> 什么是世界模型

# 符合直觉的推测

> 人工干预是否是必要的？

正文写自己的推理。
```

写法要求：

- 先写研究问题，再写论文细节。
- 先写自己的判断，再引用论文补证据。
- 每个段落只承担一个推理任务。
- 能用中文解释就用中文解释，关键术语保留英文。
- 不要只列论文标题，要说明它对当前问题的作用。

# 作者疑问的记录方式

如果是作者本人在阅读中的疑问、判断、困惑或待验证想法，用引用块保留。

```md
> 所以这个看上去很牛逼的模型有什么意义呢？
```

这类内容不要删掉，也不要改写成过度正式的论文腔。它们是研究路线形成过程的一部分。

引用块适合记录：

- 当前还没有答案的问题
- 读论文时产生的直觉
- 对论文贡献的怀疑
- 对研究方向的下一步追问
- 对概念边界的临时判断

# 论文链接写法

正文第一次提到关键论文时，直接写成普通 Markdown 链接：

```md
[World Models](https://arxiv.org/abs/1803.10122)
```

如果这篇论文在 `_链接卡片.md` 中有同名卡片，网页中点击时会打开右侧论文卡片，而不是直接离开当前页面。

匹配逻辑主要依赖：

- 正文链接文字和 `_链接卡片.md` 的二级标题一致
- 或正文链接 URL 和卡片的 `link:` 一致

因此推荐保持：

```md
[World Models](https://arxiv.org/abs/1803.10122)
```

对应：

```md
## World Models

type: 论文
date: 2018
authors: David Ha et al.
link: https://arxiv.org/abs/1803.10122
```

# 关键词链接写法

正文或论文卡片中遇到关键概念时，用空链接标记：

```md
[Latent Space]()
[POMDP]()
[VAE]()
[MDN]()
```

这些词不会跳转到外部网站，而是匹配 `_链接卡片.md` 中同名的关键词卡片。

对应写法：

```md
## Latent Space
type: 关键词

latent space 被定义为被训练成的可模拟世界演化的空间。
```

关键词卡片适合解释：

- 论文中的核心概念
- 缩写
- 模型组件
- 数学对象
- 和其他概念的区别
- 作者自己的理解和疑问

# _链接卡片.md 的总体结构

`_链接卡片.md` 由多个二级标题组成。每个二级标题就是一个卡片。

```md
## World Models

type: 论文
date: 2018
authors: David Ha et al.
link: https://arxiv.org/abs/1803.10122

### 关键点
- ...

### 模型结构
- ...

### 局限
- ...

## VAE
type: 关键词

Variational Autoencoder，变分自编码器。
```

不要在卡片里额外写 `title:`。二级标题就是卡片标题。

不要写 `aliases:`。当前默认用正文链接文字或链接地址匹配。

# 论文卡片规范

论文卡片必须写：

```md
## 论文名

type: 论文
date: 发表年份
authors: 作者
link: 论文链接
```

论文卡片正文推荐包含这些小节：

```md
### 背景
说明论文为什么出现，它试图解决什么问题。

### 关键点
- 提炼论文真正有用的判断
- 不要逐段复述 abstract
- 只保留能服务当前研究问题的内容

### 模型结构
说明模型由哪些模块组成，每个模块承担什么作用。

### 局限
- 写出论文没有解决什么
- 写出它为什么可能不适合当前目标
- 写出后续研究可能从哪里改进
```

如果有图，直接放在相关小节中：

```md
### 模型结构
![[Pasted image 20260603083343.png]]
- 视觉模块 V 使用 [VAE]() 压缩得到 latent state
- 记忆模块 M 采用 [MDN]()-RNN 结构
```

# 关键词卡片规范

关键词卡片只需要：

```md
## 关键词
type: 关键词

解释正文。
```

关键词卡片不需要 `date`、`authors`、`link`。

关键词卡片的内容应当短于论文卡片，但不能只写词典定义。应该写：

- 它在当前论文中指什么
- 它和相邻概念有什么区别
- 它为什么对当前研究问题重要
- 如果作者有疑问，用引用块保留

示例：

```md
## POMDP
type: 关键词

**Partially observable Markov decision process**
部分可观测马尔科夫决策过程。

- 普通 Markov 假设智能体能看到完整状态
- 当前问题中智能体只能看到图像，所以不能直接得到真实环境状态
- 因此需要用历史观测和动作来推断潜在状态
```

# 论文卡片中的关键词联动

论文卡片内部也可以继续写关键词链接：

```md
以往方法假设可以了解世界内部状态，但现实中只能观察图像，因此需要在更紧凑的 [Latent Space]() 中理解系统动态。
```

如果 `_链接卡片.md` 里存在：

```md
## Latent Space
type: 关键词
```

那么点击论文卡片里的 `[Latent Space]()` 也会弹出关键词卡片。

因此，论文卡片和关键词卡片不是两套孤立内容。论文卡片负责讲论文，关键词卡片负责解释论文中反复出现的概念。

# 公式写法

正文和卡片中都可以写 LaTeX 公式：

```md
$p(s_t | s_{t-1},a_{t-1})$
```

或：

```md
$$
p(s_t | s_{t-1},a_{t-1})
$$
```

公式后面必须写一句直觉解释，不要只放公式。

示例：

```md
- Transition model 状态转移模型：$p(s_t | s_{t-1},a_{t-1})$，根据上一个时间步的状态和动作来预测当前状态。
```

# 图片写法

论文结构图、模型图、关键实验图可以放进卡片。

图片写法：

```md
![[Pasted image 20260603083343.png]]
```

图片下面必须跟解释性 bullet。不要只贴图。

```md
![[Pasted image 20260603083343.png]]
- 视觉模块 V 使用 [VAE]() 压缩图像，得到 latent state z
- 记忆模块 M 让 z 受到时间影响，形成可预测的状态序列
```

# 表格写法

当研究涉及多个方向比较时，用表格组织。

推荐列：

```md
| 方向 | 研究热度 | 大致起源 | 近年进展 | 是否前沿 | 对当前想法的价值 |
|---|---|---|---|---|---|
```

表格中的论文也要写成可点击链接：

```md
[Towards Ontology Construction with Language Models](https://arxiv.org/abs/2309.09898)
```

如果该论文值得后续精读，再在 `_链接卡片.md` 中补一张论文卡片。

# 自动总结论文时的输出流程

当需要全自动总结一篇论文研究点时，按这个顺序输出：

1. 判断论文属于哪个研究问题。
2. 提取论文背景，不超过一小节。
3. 提取论文核心贡献，只保留与当前研究路线有关的点。
4. 拆出关键概念，判断哪些需要关键词卡片。
5. 拆出模型结构，说明模块之间的因果或信息流关系。
6. 提取关键公式，并写直觉解释。
7. 提取局限，尤其是对当前研究目标不够用的地方。
8. 给出当前论文在研究路线中的位置：前置基础、核心方法、对照方案、可改进对象、还是边缘参考。
9. 生成正文引用写法。
10. 生成 `_链接卡片.md` 中应追加的论文卡片和关键词卡片。

# 自动总结时的最终产物

每次总结一篇论文，至少产出两块内容。

第一块：可放进普通笔记正文的研究路线文字。

```md
# 论文精读路线

1. [World Models](https://arxiv.org/abs/1803.10122) 提出了用压缩潜表示、记忆模型和控制器构建可预测世界的基本路线。
```

第二块：可追加进 `_链接卡片.md` 的卡片。

```md
## World Models

type: 论文
date: 2018
authors: David Ha et al.
link: https://arxiv.org/abs/1803.10122

### 关键点
- ...

### 模型结构
- ...

### 局限
- ...
```

如果论文中出现需要反复解释的概念，继续追加关键词卡片：

```md
## VAE
type: 关键词

Variational Autoencoder，变分自编码器。这里重点解释它为什么被用于压缩图像观测，以及为什么它可能编码与任务无关的信息。
```

# 写作风格要求

- 使用中文解释，保留必要英文术语。
- 不要写成论文翻译。
- 不要写成百科词条。
- 不要为了完整而堆细节。
- 保留作者自己的研究疑问。
- 每篇论文都要落回“它对当前研究问题有什么用”。
- 如果论文只是相关但不核心，要明确写出“它提供背景，不是当前方案的直接答案”。
- 如果一个概念会跨多篇论文反复出现，必须抽成关键词卡片。
- 如果一个论文会被后续反复引用，必须抽成论文卡片。

# 最低合格标准

一篇合格的论文研究点笔记至少包含：

- 正文中出现一次论文链接
- `_链接卡片.md` 中有该论文的 `type: 论文` 卡片
- 至少提炼 3 个关键点
- 至少写出 1 个局限
- 至少说明 1 个和当前研究问题的关系
- 如果有核心术语，至少建 1 个 `type: 关键词` 卡片

# 不要做的事

- 不要把所有论文都写成同样模板。
- 不要只复制 abstract。
- 不要只列 Related Work。
- 不要把所有概念都塞进正文，导致正文难读。
- 不要在正文里写大段论文元数据。
- 不要给关键词卡片硬加外部链接；没有 `link:` 就不写。
- 不要删除作者疑问，除非确认它已经被后文回答。
