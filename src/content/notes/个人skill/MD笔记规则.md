## MD 笔记写作规则

写 Markdown 笔记时遵守以下规则：

1. 如果作者要求记录自己的思考过程、临时判断或提出的疑问，必须保留这些内容，并使用 Markdown 引用块记录：

   > 作者当时的思考、疑问或原始问题

2. 如果文中的模型、概念或术语来自某篇论文，优先在正文第一次出现的位置直接添加超链接。链接应绑定在模型名称、方法名称或关键术语本身上。

3. 不要默认在文章末尾堆 reference 列表。除非用户明确要求正式文献列表，否则引用应尽量融入正文。

4. 如果某个概念、论文或术语需要额外解释，但正文不适合直接展开，可以在正文中使用 `[[概念名]]` 标记，并在文末维护同名概念卡片：

   ```md
   正文里第一次提到 [[CNN]] 的时候，读者可以点击它查看右侧说明卡片。

   ## 概念卡片库

   ### [[card:CNN]]

   title: 卷积神经网络
   type: 概念
   link: https://example.com

   CNN 是一种常用于图像特征提取的神经网络结构。
   ```

   `## 概念卡片库` 放在文章末尾，方便在 Markdown 编辑器里折叠；网站渲染时不会把这个卡片库直接显示在正文末尾，而是会把它变成右侧说明卡片。

5. 如果正文里的论文、模型或概念写成 Markdown 链接，例如 `[Flamingo](https://arxiv.org/abs/2204.14198)`，并且希望点击后不直接离开当前页面，需要在该笔记本目录下维护 `_链接卡片.md` 文件。

   正文链接的显示文字应当和 `_链接卡片.md` 中的二级标题一致。网站会优先用这个二级标题匹配卡片，`link:` 只作为弹窗里的真实跳转地址和兜底匹配。

   ```md
   ## Flamingo

   type: 关键论文
   date: 2022
   authors: Jean-Baptiste Alayrac et al.
   link: https://arxiv.org/abs/2204.14198

   这里写这篇论文、模型或概念的补充说明。
   ```

   这里不需要再写 `title:` 和 `aliases:`。二级标题就是卡片标题，也是正文链接文字的主要匹配依据。
